# Schema — derived from automin (proven base) + scoped enhancements

Source: the automin Django project's `core/models.py` (working Django schema). We keep the
download→encode→archive core, drop the distribution half (the spec's out-of-scope list), and fold
automin's over-normalized models into fewer tables. Enhancements are marked **[NEW]**.

## automin model → ssanime-gui verdict

| automin model | Verdict | Notes |
|---|---|---|
| `BaseModel` (uuid, added_at, modified_at) | **KEEP** as base cols on every table | uuid is load-bearing — automin & the Wails build used it for per-item file dirs |
| `Anime` | **KEEP → `series`** | title, feed_title, alt names, poster, picture_type. **Its inline encode settings move to `encode_profiles`** (see decision 1) |
| `Feed` | **KEEP → `feeds`** | drop `upload_*` flags (distribution). Keep deinterlace/uncensored/bluray/extra_tags as filter flags |
| `SiteWatcher` | **MERGE into `feeds`** | it's just `type='scrape'` — same shape (site, url, last_check, offset, data JSON) |
| `Episode` | **KEEP → `episodes`** | the core entity (keeps automin's name). Drop all output URLs (watch/download/torrent/magnet/short_*) — those are distribution |
| `EpisodeInfo` | **MERGE into `episodes`** | release_group, original_file_size→`source_size`, new_file_size→`encoded_size`, original_(torrent/magnet)_url→source |
| `EpisodeTitle` | **FOLD → `episodes.title` text** | decided: fold (one encode per source) |
| `Screenshot` | **KEEP → `screenshots`** | decided: include in v1 as library thumbnails; generated by a post-encode ffmpeg pass (automin's `genscr`) |
| `Batch`, `BatchBundle`, `BatchFeed` | **DROP** | batch *torrent packaging* = distribution. "Download multiple" is just N `episodes` |
| `UploadSite`, `Tracker`, `TorrentFile`, `TorrentShortLink` | **DROP** | tracker uploads + torrent creation |
| `LinkSite`, `AnimeLink` | **DROP** | public-site download links |
| `SeedBox`, `QBitTorrent`, `AdflyCredential` | **DROP** | seedbox FTP, external qBit (we embed anacrolix), URL shortening |
| `Post`, `PostTopic`, `TextPage` | **DROP** | blog / public site |
| singleton pattern (SeedBox/QBit enforce one row) | **KEEP idea** | `settings` is a single-row table |

## Details automin proves that the spec got wrong / vague

- **CRF is a float, not int.** automin: `crf = FloatField(default=24.2)`. The Wails `EncodingProfile`
  used `int` CRF — wrong. x265 CRF is fractional. `encode_profiles.crf` = REAL.
- **The real encode knobs** automin tuned (per-anime): `crf 24.2`, `smartblur on`, `deblock '1,1'`,
  `psy_rd 1`, `psy_rdoq 1`, `aq_strength 1`. These become the **default profile** values, and the
  Profiles UI should expose automin's curated choice lists (deblock `-2,-2`…`2,2`; psy_rd
  1/1.25/1.5/2; psy_rdoq 1/1.5/2/3; aq_strength 0.7/0.8/0.9/1/1.25).
- **`subtype` = hardsubs | softsubs** — a real filter dimension automin tracked; keep on `episodes` and `feeds`.
- **`resolution` ∈ {1080,720,480}** integer — keep.
- **`release_group`** is tracked per episode (parsed from filename) — keep; drives naming.
- **Feed `data` JSON cache** of already-seen entries — keep; this is how a feed avoids re-enqueuing.
- **`error_message` TEXT** alongside every status — keep on `episodes` and `feeds`.
- **Source vs encoded size** are the whole point (savings) — `source_size` / `encoded_size` BIGINT.

## Proposed scoped schema (SQLite)

```
series        id, uuid, title (uniq), feed_title, alt_titles (text/json),
              season_number (int, def 1, editable),   -- for Jellyfin SxxEyy path building
              subscribed (bool),    -- auto-poll + auto-download new episodes
              favorite (bool),      -- list membership (no polling implication)
              airing_status,        -- cached AniList: NOT_YET_RELEASED|RELEASING|FINISHED|HIATUS|CANCELLED
              poster_path, poster_portrait (bool), default_profile_id → encode_profiles,
              added_at, modified_at
              -- + AniList fields (see additions): anilist_id, mal_id, romaji_title,
              --   english_title, format, status, episode_count, synonyms(json),
              --   cover_image_url, banner_image_url, season, season_year

feeds         id, uuid, series_id → series, type ('rss'|'scrape'), site, url (uniq),
              -- filter rules:
              quality (int, nullable), subtype, deinterlace, uncensored, bluray, title_regex, extra_tags,
              interval_seconds, offset_seconds, last_checked_at, seen_cache (json),
              enabled, error_message, added_at, modified_at

encode_profiles  id, uuid, name (uniq),
              builtin (bool)         -- TRUE = immutable, seeded from automin; UI can't edit/delete
              parent_id → encode_profiles (nullable),   -- inherit from a base profile
              -- knob columns are NULLABLE: NULL = inherit from parent (resolve up the chain)
              codec, crf (REAL), preset, smartblur, deinterlace, deblock,
              psy_rd (REAL), psy_rdoq (REAL), aq_strength (REAL), aq_mode (int),
              scale (int), audio, container, x265_params,
              added_at, modified_at

episodes      id, uuid, series_id → series, title, episode_no (nullable = movie/OVA/special),
              source_kind ('torrent' in v1; 'direct'|'hls' reserved), source_url, magnet, release_group,
              resolution, subtype, uncensored, bluray, published_at,
              status ('queued'|'downloading'|'downloaded'|'encoding'|'encoded'|'thumbnailing'|'archived'|'error'),
              source_path, encoded_path, source_size, encoded_size,
              profile_id → encode_profiles,            -- [NEW] which profile requested
              encoded_params_snapshot (json),          -- [NEW] exact x265 args used (reproducibility)
              downloaded_at, encoded_at,               -- [NEW] per-stage stamps for analytics
              retry_count (int),                       -- [NEW] queue retry bookkeeping
              error_message, added_at, modified_at

screenshots   id, uuid, episode_id → episodes, series_id → series, path, caption (nullable),
              ordinal (int), added_at        -- generated post-encode (automin genscr)

settings      id (always 1), download_root, encoded_root,
              cleanup_policy ('delete'|'keep'|'move'), processed_dir (for 'move'),
              naming_template (default Jellyfin: '{series}/Season {season}/{res}/{series} - S{season}E{episode}.{ext}'),
              download_backend → download_clients, default_profile_id → encode_profiles,
              concurrency_download (int), concurrency_encode (int),
              ffmpeg_path, ytdlp_path, port, doh_enabled (bool), added_at, modified_at
```

Status pipeline (drops automin's distribution stages `ups`/`upt`, keeps thumbnailing):
`queued → downloading → downloaded → encoding → encoded → thumbnailing → archived` (+ `error`).
automin equivalent: `dlfin → enc → genscr → fin`. Thumbnails are an ffmpeg pass on the *encoded*
file (N frames → `screenshots`), so `thumbnailing` sits between `encoded` and `archived`.

## Enhancements proposed [NEW] — all additive, easy to drop

1. **`episodes.profile_id` + `encoded_params_snapshot`** — automin mutated encode settings on the Anime,
   so you couldn't tell what an old encode actually used. Snapshotting the exact args makes encodes
   reproducible and lets you re-encode identically.
2. **Per-stage timestamps** (`downloaded_at`, `encoded_at`) — enables real savings/throughput analytics.
3. **`source_kind` + `retry_count`** — the Downloader-interface backend selector and queue retry
   bookkeeping (automin offloaded this to Celery + external qBittorrent; we own the queue now).
4. **Live progress (percent/speed/eta) is NOT persisted** — it's transient, pushed over SSE and held
   in memory. DB stores only durable state (status, paths, sizes) so a crash can resume cleanly
   (reset orphaned `downloading`/`encoding` → `queued`/`downloaded` on boot, per the Seanime pattern).

## Encode profile inheritance model

- **`builtin` profiles** are seeded from automin's tuned config (`crf 24.2, smartblur on, deblock
  '1,1', psy_rd 1, psy_rdoq 1, aq_strength 1`, preset slow) and are **immutable** — all knob columns
  non-NULL (a fully-specified base). The UI shows them read-only and offers "Duplicate / Customize."
- **User profiles** set `parent_id` to a builtin (or another profile) and fill **only the columns
  they override**; every other column is `NULL` = inherit.
- **Resolution:** effective config = walk `parent_id` to the root, `COALESCE` each field child→parent
  (resolved in Go; typically one hop). "Reset a knob to default" = set its column back to `NULL`.
- **`episodes.encoded_params_snapshot`** stores the *resolved* args at encode time, so editing a profile
  later never changes what an already-archived episode was encoded with.

This gives exactly: immutable automin defaults, customize-by-inheritance, override-only-what-you-need.
(Seed can include per-resolution builtins — "Automin 1080p/720p/480p" — sharing knobs, differing `scale`.)

## Sourcing (v1 scope)

- **Torrents only** in v1 — embedded `anacrolix/torrent`, the single `Downloader` backend. yt-dlp /
  direct / HLS are **deferred** (interface keeps them pluggable; not built now).
- **Original, non-remixed source files** is a sourcing-quality rule: prefer trusted original release
  groups + native resolution (we do the re-encode ourselves). Enforced via feed filter rules +
  `episodes.release_group` + an autoselect "best original release" step borrowed from Seanime.

### Two feeder kinds (a registry — `feeds.site` + `feeds.type` already model this)

automin's `core/feeders.py` is a feeder registry (`__all__ = NyaaSi, NyaaNet, Anidex, SubsPlease`).
It revealed an asymmetry we should design around:

- **Structured site feeders (e.g. SubsPlease).** automin hit the SubsPlease show page → `sid` →
  JSON API (`/api/?f=show&sid=`) → a structured `episode` dict with per-resolution `downloads`
  (`res:'1080'`, `magnet`, `torrent`). **Pre-parsed magnets per episode/resolution — low risk.**
  Keep these as first-class; they need no fuzzy matching.
- **Provider search (Nyaa/AnimeTosho/etc.).** automin's `NyaaSi`/`NyaaNet` just `feedparser.parse`d
  an RSS URL → raw titles, `magnet=None`, **no series/episode understanding** (Anidex was an unfinished
  stub). The original project leaned on SubsPlease *precisely because* matching "is this title *this
  series, this episode, this resolution, trusted group*?" from raw Nyaa was unsolved.

**The missing piece is now available:** Seanime's `habari` (release-name parser) + AniList metadata +
`autoselect` is exactly that matcher. So v1 supports **both** kinds behind one feeder-registry
interface: structured site feeds (reliable, no parsing) **and** provider search made reliable by
habari-parse → AniList-match → autoselect-best-original. Both normalize to the same
`{series, title, episode_no, resolution, release_group, magnet/torrent_url}` that creates `episodes`.

- Borrow from Seanime (`internal/torrents/...`): provider-extension **search**, **`habari`**
  release-name parsing (resolution/group/episode/batch), and **autoselect** (best torrent + right
  file in a batch). Hayase (Svelte) is a UI/UX reference only — its repo is DMCA-locked.

### Reachability — the user's ISP DNS-blocks nyaa (VERIFIED + solved with DoH)

Verified 2026-06-06 on the dev machine via a local nyaa validator:

- **nyaa.si is ISP DNS-poisoned** — the system resolver returns sinkhole `202.83.21.14` (→ TCP reset).
  But the ISP does **NOT** IP-block: resolving nyaa's *real* IP via **DoH** (`https://1.1.1.1`) and
  dialing it directly returned **75 live Frieren results**, fully parsed (Erai-raws/SubsPlease/Judas
  1080p, seeders, episode, subs). **→ DoH alone is sufficient; no proxy required.** Seanime ships
  `internal/doh` for exactly this — adopt it as the resolver for all source fetches.
- **subsplease.org works directly** (no DoH needed) — RSS parsed cleanly via gofeed + habari.
- **Hosted proxy (Hayase's method) is the fragile fallback, not the default.** Hayase's `nyaa.js`
  calls `torrent-search-api-livid.vercel.app/api/nyaasi/<q>` — but that public deploy is now
  **`DEPLOYMENT_NOT_FOUND` (dead)**. It illustrates the model (talk to a clean domain that proxies
  the blocked site) but proves you must **self-host / make the endpoint configurable** if used at all.

**Order of preference for ssanime-gui:** (1) direct fetch through a **DoH resolver** (default, proven,
zero external deps) → (2) SubsPlease direct → (3) optional configurable proxy backend for networks
that also IP-block.

The validator (`go run . -src nyaa|subsplease|proxy <query>`, `-doh` on by default) proves the
gofeed + habari + autoselect + magnet pipeline end-to-end. NOTE: it needs Go's TLS — Windows `curl`
fails these hosts with a benign `CRYPT_E_NO_REVOCATION_CHECK` SChannel quirk (`--ssl-no-revoke`), not
a real block.

### Extension system — Seanime and Hayase both have one (and we can borrow the interface)

Confirmed in the clone: Seanime has a **full JS extension system** identical in spirit to Hayase's:
- `internal/extension/hibike/torrent/types.go` defines the **`AnimeProvider` interface**:
  `Search / SmartSearch / GetTorrentInfoHash / GetTorrentMagnetLink / GetLatest / GetSettings`,
  returning `AnimeTorrent{Name, Seeders, MagnetLink, InfoHash, Resolution, ReleaseGroup,
  EpisodeNumber, IsBatch, IsBestRelease, Confirmed, ...}`. SmartSearch takes AniList/AniDB IDs +
  resolution/batch/episode/bestReleases filters — the matcher automin lacked.
- `internal/extension_repo/goja_*.go` + `external.go` + `marketplace.go` run those providers as
  **JS in goja**, fetched from a community repo/marketplace (Hayase's `ReWelp/HayasexShiru-Extensions`
  `hayase/nyaa.js` is the same shape).

**Does adopting it "massively reduce code"?** Not in v1 — the runtime itself (goja + loader +
marketplace + sandbox) is a sizeable subsystem. The real, long-term win is *maintenance*: sources
break (the dead vercel proxy, yt-dlp-style rot) and community extensions absorb that without an app
release. **Plan:** define our source layer behind the **hibike `AnimeProvider` interface now**, ship
v1 with 1–2 **native Go adapters** (SubsPlease + nyaa-via-DoH — both already validated), and add the
**goja extension runtime later** when source count/breakage justifies it. Then we can run the Seanime
hibike / Hayase JS extensions verbatim. This matches "extensible by data shape, no speculative
abstraction": the interface is cheap now; the runtime comes when it pays for itself.

## Comprehensive-v1 additions (decided 2026-06-06 — no-rush, build-it-all)

License: **GPL-3.0** (uses habari + adapts Seanime, both GPL). Four big-ticket inclusions, each with
its schema impact:

### 1. AniList metadata → enrich `series`
Add: `anilist_id`, `mal_id`, `romaji_title`, `english_title`, `format` (TV/MOVIE/OVA/ONA/SPECIAL),
`status`, `episode_count`, `synonyms` (json), `cover_image_url`, `banner_image_url`, `season`,
`season_year`. Poster can come from AniList instead of manual upload. Powers SmartSearch fuzzy match
(AniList/AniDB IDs feed the hibike `SmartSearch` filters).

### 2. Multi-resolution output → new `encoded_outputs` table
One source episode → N encoded files (1080/720/480). Don't put encoded fields on `episodes`; split:
- `episodes` keeps source + overall status; `encoded_path`/`encoded_size` columns are **dropped**
  in favour of children.
```
encoded_outputs  id, uuid, episode_id → episodes, resolution (int),
                 profile_id → encode_profiles, encoded_params_snapshot (json),
                 status ('queued'|'encoding'|'encoded'|'thumbnailing'|'archived'|'error'),
                 encoded_path, encoded_size, error_message, encoded_at, added_at, modified_at
```
`encode_profiles` gets `output_resolutions` (json int set, e.g. `[1080,720,480]`). The encode queue
fans one downloaded episode into one `encoded_outputs` row per resolution. Screenshots attach to the
episode (from the highest-res output).

### 3. goja JS extension runtime → new `extensions` + `extension_repos` tables
Reuse the MIT **hibike** `AnimeProvider` interface; run JS via goja (adapt Seanime's `extension_repo`).
```
extension_repos  id, uuid, name, url (raw index.json), enabled, last_synced_at, added_at
extensions       id, uuid, repo_id → extension_repos (nullable for local), ext_id (uniq),
                 name, type ('anime-torrent'|'manga'|...), lang ('js'), version,
                 source_url, payload (text = JS), enabled, is_builtin, settings (json),
                 added_at, modified_at
```
Seed builtins: native Go SubsPlease + Nyaa-via-DoH providers (registered through the same interface,
`is_builtin=1`, no JS payload). Hayase/Seanime JS extensions install into the same table.

### 4. External torrent-client backends → `download_clients` + `settings` selector
```
download_clients id, uuid, kind ('embedded'|'qbittorrent'|'transmission'), name,
                 host, port, username, password, enabled, is_default, settings (json), added_at
```
`settings.download_backend` picks the default; embedded anacrolix is seeded as `is_default`.
The `Downloader` interface dispatches to the selected client.

**Table count → ~10:** `series, feeds, encode_profiles, episodes, encoded_outputs, screenshots,
extensions, extension_repos, download_clients, settings`.

## Decisions (resolved 2026-06-06)

1. **Encode settings → reusable `encode_profiles` + `series.default_profile_id` + per-item
   `encoded_params_snapshot`.** Not automin's inline-on-series columns.
2. **`EpisodeTitle` → folded into `items.title` text** (+ optional `episode_no`).
3. **Screenshots → included in v1** as library thumbnails; `screenshots` table + a post-encode
   ffmpeg thumbnail pass (`thumbnailing` stage between `encoded` and `archived`).
4. **Profiles → self-referential table with `builtin` (immutable, seeded from automin) + `parent_id`
   inheritance**; NULL knob = inherit, resolved child→parent; encode snapshots resolved args.
5. **Source → torrents primary (embedded anacrolix + external clients)**; original/trusted release
   groups; yt-dlp & direct/HLS still deferred behind the `Downloader` interface. Borrow Seanime
   habari + autoselect search. DoH resolver baked in (defeats the ISP nyaa block).
6. **License → GPL-3.0.** Comprehensive build: AniList metadata, multi-resolution output, goja
   extension runtime, and external torrent-client backends all included (see additions above).
